🧩 The Problem: You're given an n×n grid where each row contains n numbers from {1, ..., 2n} (specifically the "second half" values), but one value per row is missing. Reconstruct the full permutation matrix where each value from 1 to 2n appears exactly once across all rows combined — find the missing value for each row!
From the code:
Build array p where p[i+j] = row[j-1] (flatten rows into a 1D array offset)
all_vals = {1, 2, ..., 2n}
known = set(p[2:2n+1]) (all known values)
The missing value = (all_vals - known).pop()
Place it at p[1] and print!

💡 My Approach — Set Difference for Missing Value:
The key insight: across all n rows with n values each, exactly one value from {1, ..., 2n} is missing. Find it using set difference!
Step 1: Flatten all given values into a set known 
Step 2: The missing value = {1,...,2n} - known 
Step 3: Place it correctly and output!
all_vals = set(range(1, 2*n+1))
known = set(p[2:2*n+1])
p[1] = (all_vals - known).pop()
print(*p[1:])

Examples:
n=3, rows=[[1,6,2],[6,2,4],[2,4,3]] → known={1,6,2,4,3}, missing=5 → p[1]=5 → 5 1 6 2 4 3 ✅
n=1, rows=[[2],[3],[4]] → known={2,3,4}, missing=1 → 2 1 ✅
n=1, rows=[[2],[3],[4]] → known={2,3,4}, missing=1 → 1 2 3 4 ✅

📌 Key Takeaways:
🧩 Set difference is the elegance king — all_vals - known finds missing elements in O(n) with zero loops or conditions. Always reach for sets when finding missing elements! 🎯
🧩 Flatten first, search once — by collapsing all rows into a single set, the problem reduces from "find missing per row" to "find one globally missing value". Perspective shift = simplification! ✅
🧩 .pop() on a single-element set — when you know exactly one element is missing, (all_vals - known).pop() retrieves it cleanly. One missing = one pop! 🐍
🧩 Indexing from 1 — the code uses 1-indexed arrays (p[1:]). Note the offset arithmetic p[i+j] carefully — off-by-one errors are the enemy in grid reconstruction problems! ⚠️
