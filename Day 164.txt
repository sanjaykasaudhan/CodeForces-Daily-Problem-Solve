If n == 1: survivor = 1 (trivially)
If n > 1: the survivor is always 2!
The antimedian deletion process always eliminates elements in a specific order that leaves 2 as the last survivor regardless of the initial permutation!
Why? In any permutation, value 2 is always "protected" by the antimedian property — it's never the middle element of any sorted triple containing both smaller and larger values in the right configuration. Through repeated deletions, all other values get removed before 2! 🛡️
