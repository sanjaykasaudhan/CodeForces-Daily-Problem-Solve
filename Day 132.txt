🧩 The Problem: You have a diamond-shaped grid of height 2n-1. You need to fill it with unit rhombuses (small diamond shapes). Find the number of ways to tile the diamond of size n!

💡 The Key Insight — It's Just n!
After visualising the diamond shape:
Size 1: 1 way (trivially one rhombus fills it)
Size 2: 2 ways
Size 3: 3 ways
Size n: n ways!
The pattern is simply n — the answer equals the input! 🎯
Why? The diamond of size n has exactly n "columns" of decision points, and at each one there's exactly one binary choice that propagates uniquely. The total count of distinct tilings equals exactly n.
That's it. Read n, print n! 🐍⚡

📌 Key Takeaways:
💎 Always look for the pattern first — before writing any complex tiling/DP logic, check small cases (n=1,2,3,4). The pattern here is immediately obvious: answer = n. Spot it and save yourself hours! 🔍
💎 Geometry can simplify to arithmetic — what looks like a complex combinatorial tiling problem can collapse into a single formula. Trust the math! 🧮
