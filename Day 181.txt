🎂 A Problem B with bit manipulation and a fascinating bitmask symmetry check! This one combines bitmasks, parity separation, and a beautiful feasibility condition!

🔗Problem Link: https://lnkd.in/gJTmzPm2

🧩 The Problem: Given two values a and b, find the largest n (from 1 to 41) such that it's possible to choose n bits where:
The sum of bits at even positions ≤ a OR ≤ b
The sum of bits at odd positions ≤ b OR ≤ a
(i.e., you can assign even-position bits to one constraint and odd-position bits to the other, in either orientation)
From the code: s_odd = sum of 2^i for even i (0,2,4,...), s_even = sum of 2^i for odd i (1,3,5,...).
Feasibility: (s_odd ≤ a AND s_even ≤ b) OR (s_odd ≤ b AND s_even ≤ a) — try both assignments! 🎯

💡 The Insight — Bitmask Parity Sums:
For n bits [0, 1, 2, ..., n-1]:
s_odd = sum of 2^i for even i = 1 + 4 + 16 + ... (powers of 2 at even positions)
s_even = sum of 2^i for odd i = 2 + 8 + 32 + ... (powers of 2 at odd positions)
These are geometric series:
s_odd = (4^⌈n/2⌉ - 1) / 3
s_even = 2 × (4^⌊n/2⌋ - 1) / 3
The feasibility check tries both orientations (swap a and b) and accepts if either works!
Find maximum n by trying 1 to 41 and taking the last valid n! ✅

Examples:
a=1, b=1 → n=1: s_odd=1,s_even=0 → 1≤1✅,0≤1✅ → valid; n=2: s_odd=1,s_even=2 → 1≤1,2>1 → try swap: 1≤1,2>1 → NO → ans=1 ✅
a=1, b=2 → n=2: 1≤1,2≤2✅ → valid; check n=3 → ans=2 ✅

📌 Key Takeaways:
🎂 1 << i for powers of 2 — clean and fast bit shifting to compute 2^i. Much faster than 2**i for large i! 🎯
🎂 Try both orientations — (s_odd ≤ a and s_even ≤ b) or (s_odd ≤ b and s_even ≤ a) handles the symmetric assignment. Never forget to check both directions! ✅
🎂 Bounded search with break — once can_make returns False, it's false for all larger n too (sums only grow). Break immediately! ⚡
🎂 Geometric sum insight — the parity sums grow exponentially. n=42 would exceed 10^12, so checking up to 41 is sufficient for typical constraints! 🔢
