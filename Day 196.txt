✅ Accepted on Codeforces! Problem A — Euclid, Sequence and Two Numbers | Round 1102 (Div. 2)

Reverse-engineering a sequence — Euclidean algorithm vibes with a special n=2 case! 📐🔢

🔗Problem Link: https://lnkd.in/g7NWWJzv

🧩 The Problem:
 Given a sequence b of length n generated by some process involving two numbers x and y with x = b[0] % b[1] (Euclidean step!), reconstruct x and y — or output -1 if impossible!
The sequence apparently satisfies: c[i] % c[i+1] == c[i+2] when sorted descending — a Euclidean-algorithm-like remainder chain!

💡 My Approach — Two Cases:
Case 1: n == 2
 Just sort descending and output! The two values themselves are x and y:
if n == 2:
 x, y = sorted(b, reverse=True)
 print(x, y)
 continue
Case 2: n > 2
Validity check: all elements must be distinct (len(set(b)) == n), else -1
Sort descending: c = sorted(b, reverse=True)
Verify the Euclidean chain: for each i, check c[i] % c[i+1] == c[i+2]
If valid: output c[0], c[1] (the original x, y that generate this sequence)
If any check fails: -1

📌 Key Takeaways:
📐 Euclidean algorithm = remainder chains — c[i] % c[i+1] == c[i+2] is exactly one step of the Euclidean GCD algorithm! Recognising this pattern is the entire key! 🎯
📐 Special-case small n — n == 2 has a trivially different (simpler) solution. Always check edge cases for the smallest valid input size! ✅
📐 Distinctness check via set() — len(set(b)) != n quickly verifies all elements are unique, a prerequisite for the descending chain to make sense! 🐍
📐 Sort descending, then verify chain — the largest two values are likely the "seed" x, y. Sort and check if the remainder chain holds! 🔢
