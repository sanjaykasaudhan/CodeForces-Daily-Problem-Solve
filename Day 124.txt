🧩 The Problem: Given a string consisting only of 'a' and 'b', you can change at most one character. Maximise the number of positions i where s[i] ≠ s[i+1] (adjacent characters differ) — i.e., maximise alternations! 🔀
A perfectly balanced string looks like "ababab..." or "bababa..." — every adjacent pair differs.

💡 The Key Insight — Only the Ends Matter!
The maximum possible alternations in a string of length n is n-1 (every adjacent pair differs). The only thing blocking a perfect alternating string is when the first and last character are the same!
Here's why:
A perfectly alternating string of length n must start and end with different characters if n is even, OR the same character if n is odd
But more importantly: if s[0] == s[-1], we can always change s[-1] to s[0]... wait, actually:
The real insight: If s[0] != s[-1], the string is already as balanced as possible — no change needed! If s[0] == s[-1], change the last character to match the first character's opposite → this maximises the chance of a perfect alternation near the end! 🔑
Simply put: make s[-1] = s[0] whenever they differ — and since s[0] ≠ s[-1] already means optimal, just set s[-1] = s[0] when equal!

Examples:
"b" → single char → no change → "b" ✅
"aabbbabaa" → s[0]='a', s[-1]='a' → equal → s[-1]='a' → "aabbbabaa" (unchanged, already optimal!) ✅
"abbb" → s[0]='a', s[-1]='b' → different → s[-1]='a' → "abba" ✅
"abbaab" → s[0]='a', s[-1]='b' → different → s[-1]='a' → "abbaaa" ✅

📌 Key Takeaways:
🔀 Think about boundary conditions — in alternation problems, the structure is constrained by where the string starts and ends. Focus on the endpoints, not the middle! 🎯
🔀 One change at the end propagates optimally — modifying the last character affects only one adjacency pair directly, but creates the right "boundary condition" for the entire string to be evaluated as balanced! ✅

🤯 The "Aha!" Moment: The checker accepts ANY string with maximum alternations — not a specific one. So you just need to ensure the endpoints create the right parity. Make s[-1] = s[0] and let the rest of the string speak for itself! 🔑
