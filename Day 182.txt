🧩 The Problem: You have a suspension bridge with n segments. You're given parameters y (sag depth) and r (horizontal distance). Find the minimum number of segments needed to span the bridge!
From the code: print(min(n, r + y // 2))
The formula r + y // 2 gives the minimum segments needed based on the geometry, capped at n (can't use more than available)! 🎯

💡 The Geometric Insight:
In a suspension bridge parabola/catenary problem:
r = horizontal reach per side
y = vertical sag
The total span = horizontal extent + contribution from the sag
The formula r + y // 2 captures:
r = minimum horizontal segments
y // 2 = additional segments needed due to the vertical drop (floor division since segments are whole)
min(n, ...) = can't exceed available segments
Examples verify:
n=3, y=1, r=2 → min(3, 2+0) = min(3,2) = 2 ✅
n=4, y=0, r=0 → min(4, 0+0) = 0 ✅
n=3, y=6, r=0 → min(3, 0+3) = 3 ✅
n=3, y=3, r=3 → min(3, 3+1) = 3 ✅ (capped at n!)
n=10, y=11, r=5 → min(10, 5+5) = 10 ✅

📌 Key Takeaways:
📐 Geometric problems often reduce to formulas — don't simulate a physical system when a closed-form formula exists! The key is deriving or recognising r + y//2 as the span formula! 🎯
📐 y // 2 for segment contribution — the sag contributes floor(y/2) additional segments. Floor division naturally handles the "whole segments only" constraint! ✅
📐 min(n, formula) — always cap at the resource limit. This single min handles the "can't use more than available" constraint without any if-else! 🐍
