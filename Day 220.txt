🚀 Codeforces Accepted | Fun Permutation 🎲💻

Today's problem, "Fun Permutation," looked like a construction problem with GCD conditions, but the key insight was to transform the given permutation into its complement permutation.

🔗Problem Link: https://lnkd.in/ge7NHKZg

💡 My Approach
🔹 Step 1: Observe the Pattern
We are given a permutation p and need to construct another permutation q such that:
gcd(p[i] + q[i], p[i+1] + q[i+1]) ≥ 3
for every pair of adjacent indices.
Instead of searching for a valid permutation through brute force, I looked for a mathematical relationship.
🔹 Step 2: Construct the Complement
For every element in the permutation:

q[i] = n + 1 - p[i]

This maps:
1 → n
2 → n−1
3 → n−2
...
n → 1
Since p is a permutation, q is also a valid permutation.
🔹 Step 3: Verify the Condition
For every position:
p[i] + q[i]
= p[i] + (n + 1 − p[i])
= n + 1

Every position has exactly the same sum.
Therefore, every pair of adjacent sums is identical:
gcd(n + 1, n + 1) = n + 1

Since n + 1 ≥ 3, the required GCD condition is always satisfied.

⚙️ Complexity Analysis
Time Complexity: O(n) per test case
Space Complexity: O(n)
Only one linear traversal is needed to construct the new permutation.

✅ Mathematical Construction

✅ Linear-Time Solution

Competitive programming often rewards finding the hidden mathematical symmetry instead of trying numerous possibilities. Here, constructing the complement permutation ensures that every position has the same sum, making the GCD condition hold automatically. 🚀
