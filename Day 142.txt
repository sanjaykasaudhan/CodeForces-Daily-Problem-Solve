✅ Accepted on Codeforces! Problem C — Sending Messages | Round 920 (Div. 3)

A Problem C with a greedy battery simulation — clean, practical, and satisfying! 📱⚡

🔗Problem Link: https://lnkd.in/geW_FMzm

🧩 The Problem: You have a phone with initial battery charge f. The phone loses a charge per second while active and b charge per second while in sleep mode. You need to send n messages at specific timestamps m[1], m[2], ..., m[n].
Between messages, you can put the phone to sleep to save battery. But waking up from sleep costs b charge per wakeup. Can you send all messages without the battery dying (charge > 0 at each send)? 🎯

💡 My Approach — Greedy Gap-by-Gap Simulation:
For each consecutive pair of message timestamps, compute the gap = msg - prev. Then decide the cheapest way to wait:
Stay awake: costs gap * a charge
Sleep and wake: costs b charge (wakeup cost only — sleep itself is free between messages!)
Take the minimum of the two: cost = min(gap * a, b)
Subtract the cost from current charge. If charge ever drops to 0 or below → NO! Otherwise after all messages → YES! ✅

Examples:
n=3, f=1, a=1, b=5, msgs=[3] → gap=3, cost=min(3,5)=3, charge=1-3=-2 → NO ✅
n=7, f=21, a=1, b=3, msgs=[4,6,10,13,17,20,26] → simulate each gap → YES ✅
n=1, f=1000000000, msgs=[1000000000] → gap=10⁹, cost=min(10⁹*a, b) → YES ✅

📌 Key Takeaways:
📱 Think per-interval, not globally — instead of planning the entire journey at once, make the locally optimal decision for each gap. Greedy per-interval works because decisions don't cascade! ✅
📱 min(stay_awake, sleep+wake) — this single comparison captures the entire decision. If staying awake is cheaper than the wakeup penalty, stay awake. Otherwise sleep! 🎯
📱 charge <= 0 is the kill condition — the phone dies if charge hits 0 OR goes negative. Make sure to check <= 0, not just < 0! ⚠️
📱 prev = 0 initialisation — the phone starts at time 0, so the first message's gap is measured from 0. Always initialise "previous time" correctly! 🕐

🤯 The "Aha!" Moment: Between any two events, you have exactly one binary choice: stay awake (cost = gap × a) or sleep (cost = b for waking up). Take the cheaper one each time. Greedy per gap = globally optimal because past decisions don't affect future cost functions! 🔑
